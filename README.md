# Secure Messenger - Максимально безопасный мессенджер

Это полная реализация архитектуры безопасного мессенджера с использованием современных технологий криптографии, биометрической аутентификации и децентрализованной P2P сети.

## Ключевые особенности

### Безопасность
- **End-to-End шифрование** с использованием Signal Protocol для личных чатов
- **MLS (Message Layer Security)** для групповых чатов
- **Post-Quantum криптография** (Kyber/Dilithium) для защиты от квантовых компьютеров
- **Zero Knowledge Proofs** для анонимности и приватности
- **Биометрическая аутентификация** без передачи данных на сервер

### Приватность
- **DID (Decentralized Identity)** для самостоятельного управления идентичностью
- **Tor/I2P интеграция** для анонимного обмена сообщениями
- **Cover traffic** для противодействия анализу трафика
- **Forward Secrecy** для всех сообщений

### Модерация и соответствие законам
- **Мультиподписная система** для контролируемой деанонимизации
- **Прозрачная система жалоб** с криптографическими доказательствами
- **Аудит-журнал** всех модерационных действий
- **Threshold encryption** для защиты от злоупотреблений

## Архитектура

Проект использует гексагональную архитектуру (Ports & Adapters) с четкими границами между слоями:

```
app/
├── domain/                 # Бизнес-логика и правила
│   ├── chat/              # Модели чатов и сообщений
│   ├── identity/          # DID, биометрия, пользователи
│   ├── moderation/        # Жалобы, мультиподписи
│   └── shared/            # Общие типы и события
├── application/           # Сценарии использования
│   ├── usecases/         # Use Cases
│   ├── services/         # Сервисы приложения
│   └── ports/            # Интерфейсы
├── infrastructure/       # Адаптеры и внешние зависимости
│   ├── crypto/           # Криптографические адаптеры
│   ├── p2p/              # P2P сетевые адаптеры
│   ├── biometrics/       # Биометрические адаптеры
│   └── storage/          # Хранилище данных
└── frontend/             # React UI
    ├── components/       # UI компоненты
    ├── pages/           # Страницы приложения
    └── hooks/           # React хуки
```

## Использование

### Регистрация

1. Заполните профиль (имя, описание, аватар)
2. Настройте биометрическую аутентификацию
3. Сохраните код восстановления из 12 слов
4. Завершите регистрацию

### Аутентификация

- **Биометрия**: Используйте отпечаток пальца, Face ID или другие биометрические данные
- **Код восстановления**: Введите 12 слов в случае проблем с биометрией

### Создание чатов

- **Личный чат**: Добавьте пользователя по DID или найдите в сети
- **Групповой чат**: Создайте группу и пригласите участников
- **Канал**: Создайте канал для односторонней рассылки

### Отправка сообщений

- **Текст**: Обычные текстовые сообщения
- **Голосовые сообщения**: Запись и воспроизведение голоса
- **Файлы**: Обмен зашифрованными файлами
- **Реакции**: Эмоции на сообщения

## Безопасность

### Криптографические гарантии

- **Signal Protocol**: Double Ratchet, Forward Secrecy
- **MLS**: Tree-based key agreement, efficient group messaging
- **Post-Quantum**: Защита от квантовых атак
- **ZK-SNARKs**: Анонимность без потери функциональности

### Биометрическая безопасность

- Данные обрабатываются только локально
- Хранятся только хеши признаков
- Nullifier предотвращает дублирование аккаунтов
- ML модели работают в браузере

### Сетевая безопасность

- Tor/I2P для анонимности
- Cover traffic против анализа трафика
- WebRTC с STUN/TURN для NAT traversal
- Relay серверы не видят содержимое

## Модерация

### Система жалоб

1. Пользователь подает жалобу с доказательствами
2. Жалоба подписывается криптографически
3. Модераторы рассматривают жалобу
4. При необходимости создается мультиподписный запрос

### Мультиподписи

- Требуется согласие нескольких модераторов
- Threshold signatures для доступа к данным
- Полная прозрачность и аудит
- Защита от злоупотреблений

### Деанонимизация

- Только по решению модераторов
- Требует веских оснований
- Логируется в аудит-журнале
- Обратимость через суд

```typescript
// Регистрация
const registerUseCase = container.get<RegisterUserUseCase>('RegisterUserUseCase');
const result = await registerUseCase.execute({

  avatar: avatarData

// Отправка сообщения

const result = await sendMessageUseCase.execute({
  chatId: 'chat-id',

    data: new TextEncoder().encode('Hello!')
  }
});
```

## 🐛 Известные проблемы

### WebRTC в некоторых браузерах
- **Проблема**: Safari может блокировать WebRTC соединения
- **Решение**: Используйте relay серверы или Tor/I2P

### Биометрия на старых устройствах
- **Проблема**: WebAuthn может не поддерживаться
- **Решение**: Fallback на код восстановления

### IndexedDB лимиты
- **Проблема**: Ограничения на размер хранилища
- **Решение**: Периодическая очистка или облачный бэкап
